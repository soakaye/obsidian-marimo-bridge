# 実装計画: コンテキストメニューへのノートブック作成機能の追加

**インプット**: `/specs/005-context-menu-create-notebook/spec.md` の機能仕様書

## 概要

本機能の目的は、Obsidianのファイルエクスプローラ内のファイルまたはフォルダを右クリックした際に表示されるコンテキストメニューに、「Create new marimo notebook」という項目を追加することです。これにより、ユーザーは特定のフォルダ（またはルートディレクトリ）内に新しい marimo ノートブック（`.py` ファイル）を即座に作成し、エディタで開くことができるようになります。

技術的なアプローチとして、既存の `file-menu` イベントハンドラーを拡張し、クリックされた対象オブジェクト（`TAbstractFile`）がフォルダであるかファイルであるかに応じて、作成先ディレクトリを動的に解決した上で、既存のノートブック作成ロジック（`createNotebook`）を呼び出します。

## 技術的コンテキスト

**言語/version**: TypeScript 5.4+ / ES2022

**主要な依存関係**: Obsidian API (特に `file-menu` イベント、`TFolder`、`TFile`、`normalizePath`)

**ストレージ**: Obsidian ローカルファイルシステム (新規 `.py` ファイルの作成)

**テスト**: Obsidian デスクトップ環境における手動検証

**対象プラットフォーム**: デスクトップ (Obsidian Desktop のみ)

**プロジェクトタイプ**: desktop-app (Obsidian プラグイン)

**パフォーマンス目標**: メニュー表示および新規ノートブックの作成・エディタでのオープン処理が200ms未満で完了すること。

**制約事項**: 特になし（Obsidian 標準の API を使用するため安全に動作）
