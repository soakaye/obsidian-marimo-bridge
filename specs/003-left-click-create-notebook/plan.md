# 実装計画: リボンアイコンの左クリックメニューによるノートブック作成機能の追加

**ブランチ**: `003-left-click-create-notebook` | **作成日**: 2026-06-19 | **仕様書**: [spec.md](file:///C:/Users/asuzuki.SSL/Documents/obsidian/working/.obsidian/plugins/obsidian-marimo-bridge/specs/003-left-click-create-notebook/spec.md)

**インプット**: `/specs/003-left-click-create-notebook/spec.md` の機能仕様書

## 概要

本機能の目的は、marimo-bridge プラグインのリボンアイコンを左クリックした際の挙動をカスタマイズ可能にすることです。
デフォルトでは、リボンアイコンをクリックすると即座に marimo のホームダッシュボードが開きます。この機能では、プラグインの設定タブに「Enable ribbon left-click menu」というトグルオプションを追加します。この設定が有効な場合、リボンアイコンを左クリックしたときに Obsidian のネイティブ `Menu` を表示し、「Open marimo home」と「Create new marimo notebook」を選択できるようにします。無効な場合（デフォルト）は、現在と同様にホームダッシュボードを直接開きます。

## 技術的コンテキスト

**言語/バージョン**: TypeScript 5.4+ / ES2022

**主要な依存関係**: Obsidian API (特に `Menu` と `addRibbonIcon`)

**ストレージ**: Obsidian ローカルデータ (プラグイン設定の `saveData` / `loadData` を介した `showRibbonMenu` トグル状態の保存)

**テスト**: Obsidian デスクトップ環境での手動検証

**対象プラットフォーム**: デスクトップ (Obsidian Desktop のみ、モバイルは非サポート)

**プロジェクトタイプ**: desktop-app (Obsidian プラグイン)

**パフォーマンス目標**: リボンアイコンをクリックした際、コンテキストメニューが即座（100ms未満）に表示されること

**制約事項**: デスクトップ専用の依存関係 (Obsidian `Menu` API およびワークスペースリーフはデスクトップに依存)

**規模/スコープ**: 影響は大きいが複雑度は低い変更。`src/main.ts` および `src/settings.ts` のみに限定される
