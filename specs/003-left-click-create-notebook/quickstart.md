# クイックスタートガイド (Quickstart) - リボンアイコンの左クリックメニューによるノートブック作成機能の追加

**作成日**: 2026-06-19

## 概要
本ガイドは、開発環境における本機能のビルド、起動、および動作確認の手順を説明します。

## 開発環境のセットアップ

### 1. 依存関係のインストール
プロジェクトのルートディレクトリで以下を実行して、必要なパッケージをインストールします。

```bash
npm install
```

### 2. 開発サーバーの起動 (esbuild watch)
コードの変更を自動検知してビルドするために、次のコマンドを実行します。

```bash
npm run dev
```

### 3. プラグインの有効化
Obsidian を起動し、`Settings` > `Community plugins` から `marimo Bridge` を有効化します。（ローカル開発環境でビルドされた `main.js`, `manifest.json`, `styles.css` がプラグインフォルダに配置されている必要があります）。

## 機能の検証手順

### シナリオ 1: デフォルト挙動の確認 (メニュー無効)
1. Obsidian のプラグイン設定を開きます。
2. 「Enable ribbon left-click menu」トグルが **オフ** であることを確認します。
3. Obsidian の左側リボンにある marimo アイコンをクリックします。
4. **結果**: メニューが表示されず、即座に marimo のホームダッシュボード（`marimo` ビュー）が新しいタブで開くことを確認します。

### シナリオ 2: メニューの表示と動作確認 (メニュー有効)
1. Obsidian のプラグイン設定を開きます。
2. 「Enable ribbon left-click menu」トグルを **オン** にします。
3. 左側リボンの marimo アイコンをクリックします。
4. **結果**: マウスクリック位置にコンテキストメニューが表示されることを確認します。
5. メニューに以下の 2 項目が表示されていることを確認します：
   - 「Open marimo home」（アイコン: `notebook-pen`）
   - 「Create new marimo notebook」（アイコン: `plus`）
6. 「Open marimo home」をクリックし、marimo のホームダッシュボードが開くことを確認します。
7. 再度メニューを表示し、「Create new marimo notebook」をクリックし、新規ノートブック（例: `untitled_marimo.py`）がアクティブフォルダ（またはルートフォルダ）に作成され、エディタで開くことを確認します。
